# Linux SysAdmin Lab

> **Proyecto personal de formación práctica en Linux System Administration, diseñado para simular el proceso de onboarding de un Linux System Administrator Trainee dentro de un equipo de Infraestructura.**

## 📖 Sobre este proyecto

Este repositorio documenta mi proceso de aprendizaje para convertirme en **Linux System Administrator**, con el objetivo de construir una base sólida que posteriormente me permita evolucionar hacia roles como:

* Linux System Administrator
* Infrastructure Engineer
* DevOps Engineer
* Platform Engineer
* Cloud Engineer

A diferencia de un repositorio de apuntes tradicional, este proyecto está diseñado como un **bootcamp interno**, donde cada módulo representa una etapa de formación similar a la que podría recibir un administrador de sistemas junior durante sus primeros meses en una empresa.

Cada laboratorio busca resolver problemas reales, documentar procedimientos, automatizar tareas y desarrollar hábitos de trabajo profesionales.

---

# 🎯 Objetivos

Este laboratorio tiene cinco objetivos principales:

* Aprender administración de sistemas Linux desde una perspectiva práctica.
* Documentar procedimientos técnicos de forma clara y reutilizable.
* Automatizar tareas repetitivas mediante Bash y scripting.
* Construir un portfolio técnico demostrable.
* Prepararme para entrevistas técnicas de Linux SysAdmin, Infrastructure y DevOps.

Cada Sprint incluye:

* Contexto

* Checkpoints
* Laboratorios

* Retrospectiva

# 🎓 Metodología

El aprendizaje sigue un enfoque basado en problemas.

Cada Sprint propone un escenario inspirado en tareas reales de Infraestructura.

En lugar de estudiar comandos de forma aislada, el objetivo es aprender a utilizarlos para resolver situaciones concretas.

Cada Sprint se desarrolla siguiendo el siguiente ciclo:

```text
Leer

↓

Comprender

↓

Implementar

↓

Romper

↓

Investigar

↓

Resolver

↓

Documentar

↓

Reflexionar
```

# 📅 Sprint actual

## Módulo 01 — Linux Fundamentals

### Sprint 01 — Onboarding de un nuevo equipo de desarrollo

**Escenario**

Un nuevo equipo de desarrollo se incorpora a la empresa.

Infraestructura debe preparar un servidor Linux compartido donde almacenarán documentación, logs, backups y archivos de trabajo.

Como Linux SysAdmin Trainee, participarás en la preparación inicial del servidor realizando tareas similares a las que recibirías durante un proceso real de onboarding.

Durante este Sprint se trabajarán los siguientes temas:

* Filesystem
* Navegación
* Organización de directorios
* Gestión de archivos
* Owners
* Groups
* Permisos
* Búsqueda de archivos
* Visualización de contenido
* Pipes
* Redirecciones

### Plan del Sprint

| Día   | Objetivo                                |
| ----- | --------------------------------------- |
| Día 1 | Conocer la estructura del servidor      |
| Día 2 | Organización de documentación y backups |
| Día 3 | Investigación de logs                   |
| Día 4 | Auditoría de permisos                   |
| Día 5 | Validación final y entrega del servidor |

El detalle completo del Sprint puede consultarse en:

`bootcamp/01-linux-fundamentals/Sprint-01/README.md`
