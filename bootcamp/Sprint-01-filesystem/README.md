# Sprint 01 — Filesystem Fundamentals

> **Módulo:** Linux Fundamentals  
> **Duración estimada:** 1 semana  
> **Dificultad:** ⭐☆☆☆☆

---

# Objetivo del Sprint

Durante este Sprint asumirás el rol de **Linux System Administrator Trainee** dentro de un equipo de Infraestructura.

El objetivo será familiarizarte con la estructura del sistema de archivos Linux, aprender a navegar el sistema con soltura y desarrollar buenas prácticas de organización y documentación.

Todas las actividades están diseñadas para utilizar únicamente herramientas básicas disponibles en una instalación estándar de Linux.

---

# Escenario

La empresa acaba de incorporar un nuevo equipo de desarrollo que trabajará sobre una aplicación interna.

El equipo de Infraestructura recibió la tarea de preparar un servidor Linux compartido donde los desarrolladores almacenarán documentación, logs, backups y archivos de trabajo.

Como nuevo integrante del equipo, participarás en las tareas iniciales de preparación del servidor bajo la supervisión de un administrador con más experiencia.

Durante esta semana recibirás pequeñas tareas operativas similares a las que podría afrontar un Linux SysAdmin durante sus primeros días de trabajo.

---

# Competencias que se trabajan

Al finalizar este Sprint deberías sentirte cómodo con:

- Navegación por el filesystem.
- Organización de directorios.
- Gestión básica de archivos.
- Visualización de contenido.
- Búsqueda de archivos.
- Comprensión de permisos, owners y grupos.
- Uso de pipes y redirecciones.
- Documentación técnica de procedimientos.

---

# Comandos principales

Durante este Sprint se utilizarán principalmente los siguientes comandos:

| Categoría | Comandos |
|-----------|-----------|
| Navegación | `pwd`, `cd`, `ls`, `tree` |
| Archivos | `mkdir`, `touch`, `cp`, `mv`, `rm` |
| Visualización | `cat`, `less`, `head`, `tail` |
| Búsqueda | `find`, `grep`, `wc` |
| Permisos | `chmod`, `chown`, `ls -l` |
| Redirecciones | `>`, `>>`, `<`, `|` |

# Laboratorio

Al finalizar los cinco días se realizará un laboratorio integrador donde deberás resolver un escenario utilizando todos los conocimientos adquiridos durante el Sprint.

El objetivo será trabajar con la menor cantidad posible de consultas externas, utilizando la documentación generada durante la semana como referencia principal.

---

# Criterios de aprobación

El Sprint se considerará completado cuando puedas:

- Navegar por el sistema sin depender constantemente de apuntes.
- Localizar archivos y directorios relevantes con rapidez.
- Organizar correctamente una estructura de directorios.
- Analizar archivos de texto utilizando herramientas básicas.
- Comprender y modificar permisos básicos de archivos y directorios.
- Utilizar pipes y redirecciones para generar información útil.
- Documentar de forma clara las tareas realizadas.

## Resultado esperado

Más allá de aprender comandos, el objetivo de este Sprint es comenzar a desarrollar la forma de pensar de un administrador de sistemas: observar el entorno, investigar, validar cambios, documentar procedimientos y trabajar de manera ordenada y reproducible.

Este enfoque servirá como base para los siguientes módulos del Bootcamp, donde se incorporarán tareas de administración de usuarios, procesos, servicios, redes y automatización.
