# Bootcamp

Este directorio contiene el programa de formación práctica del laboratorio.

El bootcamp está organizado en **módulos**, y cada módulo se divide en **Sprints** que simulan tareas y escenarios similares a los que podría afrontar un Linux System Administrator Trainee durante su proceso de onboarding en un equipo de Infraestructura.

El objetivo no es memorizar comandos, sino desarrollar habilidades de resolución de problemas mediante laboratorios prácticos, documentación técnica y reflexión sobre el trabajo realizado.

---

# Estructura

Cada módulo aborda un área específica de conocimiento.

```text
01-linux-fundamentals/
│
├── README.md
│
├── Sprint-01-filesystem/
├── Sprint-02-file-management/
├── Sprint-03-permissions/
├── Sprint-04-text-processing/
└── Sprint-05-linux-challenge/
```

Cada Sprint mantiene la misma organización:

```text
Sprint-01-filesystem/

├── README.md
├── Day-01.md
├── Day-02.md
├── Day-03.md
├── Day-04.md
├── Day-05.md
├── Lab.md
└── Retrospective.md
```

---

# ¿Qué contiene un Sprint?

| Archivo | Descripción |
|----------|-------------|
| `README.md` | Presenta el contexto, los objetivos y los entregables del Sprint. |
| `Day-XX.md` | Documenta el trabajo realizado durante cada jornada. |
| `Lab.md` | Laboratorio integrador donde se aplican los conocimientos del Sprint. |
| `Retrospective.md` | Reflexión final sobre el aprendizaje, dificultades encontradas y próximos pasos. |

---

# Criterios de finalización

Un Sprint se considera completado cuando:

- Se realizaron todas las tareas propuestas.
- Se completó el laboratorio integrador.
- Se documentó el trabajo realizado.
- Se registró una retrospectiva con los aprendizajes obtenidos.

---

Cada módulo incrementa progresivamente la dificultad y está pensado para construir una base sólida antes de avanzar hacia tecnologías de automatización, contenedores, infraestructura como código y cloud.
